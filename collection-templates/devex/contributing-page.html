Schritt fuer Schritt

Vom Fork zum Pull Request in sechs einfachen Schritten.

1
Fork
Erstellen Sie einen Fork des {{BRAND_NAME}}-Repositorys auf GitHub. Damit haben Sie Ihre eigene Kopie zum Arbeiten.
2
Clone
Klonen Sie Ihren Fork lokal und installieren Sie die Abhaengigkeiten mit npm install.
3
Branch
Erstellen Sie einen Feature-Branch: git checkout -b feature/mein-feature.
4
Code
Nehmen Sie Ihre Aenderungen vor. Halten Sie sich an die Code-Stilrichtlinien und schreiben Sie klaren, dokumentierten Code.
5
Test
Fuehren Sie die Tests aus: npm test. Fuegen Sie Tests fuer neue Funktionen hinzu. Alle Tests muessen bestehen.
6
Pull Request
Oeffnen Sie einen PR gegen den main-Branch. Beschreiben Sie Ihre Aenderungen und verlinken Sie verwandte Issues.

Code-Stilrichtlinien

Konsistenter Code erleichtert die Zusammenarbeit und Wartung.

TypeScript Strict Mode
Alle Dateien verwenden TypeScript mit strict: true. Vermeiden Sie any-Typen. Definieren Sie explizite Rueckgabetypen fuer oeffentliche Funktionen.
ESLint und Prettier
Fuehren Sie npm run lint vor jedem Commit aus. Die Konfiguration wird automatisch uebernommen. Verwenden Sie 2 Leerzeichen zur Einrueckung.
Test-Abdeckung
Mindestens 80% Test-Abdeckung fuer neuen Code. Verwenden Sie Vitest fuer Unit-Tests und Playwright fuer E2E-Tests.

Commit-Nachricht Format

Wir verwenden Conventional Commits fuer klare und konsistente Commit-Nachrichten.

# Format
<typ>(<bereich>): <beschreibung>

# Typen
feat - Neue Funktion
fix - Fehlerbehebung
docs - Dokumentation
style - Formatierung (kein Code-Aenderung)
refactor - Code-Umstrukturierung
test - Tests hinzufuegen/aendern
chore - Build-Prozess / Werkzeuge

# Beispiele
feat(events): Event-Filterung nach Typ hinzufuegen
fix(api): Race Condition bei Stream-Erstellung beheben
docs(readme): Installationsanleitung aktualisieren
